# CallFlow v0.2.0-beta.1

Beta release notes for the current public repository state.

Direct Windows installer:

- https://github.com/GaboEI/callflow/releases/download/v0.2.0-beta.1/CallFlow.Setup.0.2.0-beta.1.exe

## Summary

CallFlow is now presented as a beta-stage, local-first Windows desktop app for call-center operators. The core local workflows are functional, and the repository has already been validated on Windows 11 VM testing with a successful installer and clean onboarding pass.

## Added

- Beta-stage repository wording and status.
- Updated public documentation for recruiter-facing review.

## Changed

- Version bumped to `0.2.0-beta.1`.
- README, changelog, and public description now frame CallFlow as a beta application instead of an early prototype.
- Existing alpha validation is kept as historical context.

## Fixed

- Onboarding flow validation now blocks progression cleanly and shows legal feedback when terms are not accepted.
- Onboarding legal text is now exposed through an explicit expandable disclosure before acceptance.
- Reconfiguration keeps already accepted legal terms locked and visible, including acceptance metadata in About.
- Onboarding statistics timezone options refresh from the active timezone set.
- Settings now include a controlled local-data erase action that clears the app's userData store and restarts the app.

## Windows validation

- First successful Windows NSIS installer validation completed.
- Fresh onboarding was tested on Windows after clearing `%APPDATA%\\CallFlow`.
- Terms/privacy blocking was confirmed from clean local app data.
- Windows build generation succeeded with `npm run dist:win` during the first successful Windows validation cycle.

## Beta official hardening pass

- EULA placeholder replaced with beta installer terms (`WINDOWS_INSTALLER_EULA.txt`).
- QA artifact info added: file name, size (100 MB), SHA-256, build date, and architecture from GitHub Releases.
- Clipboard call ID sanitization hardened: only alphanumeric characters, hyphens, underscores, dots, and spaces are allowed.
- Local data erase now requires a main-process confirmation token (`CONFIRM_CLEAR_ALL`); renderer UI alone cannot trigger deletion.
- Backup import now uses an internal modal instead of `window.confirm`, consistent with the erase confirmation pattern.
- External link policy restricted to HTTPS only; all other protocols are blocked and logged.
- Generic clipboard read API replaced with `readClipboardCallId` that sanitizes input in the main process.
- `npm run check` replaced with auto-discovery script (`scripts/check-syntax.js`).
- Security policies documented in `SECURITY.md` for imports, clipboard, external links, and destructive operations.

## Known limitations

- AI Chat is still a placeholder.
- Installer beta terms are included. Privacy and terms drafts remain under review before a stable public release.
- Beta UX and release polish continue to be refined.

## Installer notes

- Windows NSIS build target remains the primary packaging path.
- Portable Windows output is still supported for local testing.
